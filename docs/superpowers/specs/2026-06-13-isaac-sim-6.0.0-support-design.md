# Isaac Sim 6.0.0 Support — Design

## Goal

Make `isaacsim-mcp-server` run cleanly against Isaac Sim 6.0.0 while keeping 5.1.0 working. The existing `IsaacAdapterV5` already isolates version-specific calls behind an ABC, so most of the work is targeted: detect the runtime version, suppress / handle expected deprecation paths, and patch the calls that actually changed semantics.

## Context: what changed in 6.0.0

Isaac Sim 6.0.0 reorganizes the Python namespace around two ideas:

1. **A new "Core Experimental" API** (`isaacsim.core.experimental.{utils,prims,objects,actuators,materials}`) built on `warp` and `omni.physics.tensors`, intended to be the long-term replacement for `isaacsim.core.{api,prims,utils}`.
2. **A new `SimulationManager` singleton** (`isaacsim.core.simulation_manager`) that owns physics backend selection (`physx` vs `newton`) and supersedes `World` / `SimulationContext`.

Critically, the 5.1 extensions still ship in 6.0.0 under `extsDeprecated/`:

| Extension used by V5 adapter | Status in 6.0.0 |
| --- | --- |
| `isaacsim.core.api` (`World`, `SimulationContext`) | Deprecated, still functional |
| `isaacsim.core.prims` (`SingleArticulation`, `SingleXFormPrim`) | Deprecated, still functional |
| `isaacsim.core.utils.{prims,stage,types}` | Deprecated, still functional |
| `isaacsim.sensors.camera.Camera` | Deprecated, still functional |
| `isaacsim.sensors.rtx.LidarRtx` | Deprecated, still functional |
| `isaacsim.storage.native.get_assets_root_path` | **Not deprecated** — unchanged |
| `omni.kit.commands.execute("URDFCreateImportConfig"/"URDFParseFile"/"URDFImportRobot")` | Registered, emits `carb.log_warn` per call |
| `omni.physx.get_physx_interface().get_rigidbody_transformation()` | **Unchanged** (verified in `omni.physx-110.1.11` `.pyi`) |
| `omni.kit.commands.execute("DeletePrims"/"CopyPrim"/"CreatePrim")` | Unchanged (standard kit commands) |
| `omni.usd`, `omni.timeline`, `omni.kit.app`, `omni.client`, `pxr.*` | Unchanged |

Each deprecated extension prints a one-line warning via `isaacsim.core.deprecation_manager` on first import. Nothing actually broke in the V5 adapter — it should run on 6.0.0 today, just noisy.

## Scope decision: minimal V6 adapter, not a rewrite

The "right" long-term direction is to migrate to `isaacsim.core.experimental.*`, but that API has fundamentally different semantics (batched `wp.array` over multi-prim views vs. single-prim `numpy` wrappers). A full rewrite is a bigger project than version support and would also chase still-shifting APIs in a 6.0 release tagged `-rc.59`.

This spec covers only:

1. **Runtime version detection** in `adapters/__init__.py:get_adapter()`.
2. **A new `IsaacAdapterV6(IsaacAdapterV5)`** that overrides only the calls where 5.1 behavior differs from 6.0 behavior. Today that is:
   - `import_urdf` — migrate to `URDFImporter` / `URDFImporterConfig` (new public class) to avoid log spam and prepare for command removal.
   - `_ensure_physics_world` — prefer `SimulationManager.initialize_physics()` when present; fall back to `World.instance()` (the V5 path).
3. **Suppress the expected deprecation noise** from the remaining deprecated imports by routing them through a single helper that filters one specific carb log channel during import. Out of scope if it turns out the warnings only fire once at startup (then we just document them).
4. **A smoke test matrix** in `tests/` that imports each adapter against its expected runtime; can be gated behind an `ISAACSIM_VERSION` env var since CI won't have Isaac Sim installed.

Everything else (cameras, lidar, articulations, world, prim creation) keeps using the 5.1-deprecated path until a follow-up "experimental migration" project. That follow-up can land per-subsystem (camera, then articulation, etc.) without blocking 6.0 release support.

## Architecture

```
adapters/
├── base.py              (unchanged — ABC)
├── v5.py                (unchanged — 5.1.0)
├── v6.py                (NEW — IsaacAdapterV6(IsaacAdapterV5))
└── __init__.py          (UPDATED — get_adapter() with version detection)
```

`get_adapter()` resolves the runtime version once at extension startup:

```python
def get_adapter() -> IsaacAdapterBase:
    major = _detect_isaacsim_major_version()  # returns 5 or 6
    if major == 6:
        from .v6 import IsaacAdapterV6
        return IsaacAdapterV6()
    from .v5 import IsaacAdapterV5
    return IsaacAdapterV5()
```

Version detection reads `isaacsim.core.version.get_version()` if available (present in both 5.1 and 6.0), with a fallback to reading the install's `VERSION` file via `carb.tokens` (e.g. `${app}/../VERSION`). On any failure, default to V5 — the safer choice since V5 also runs on 6.0 with warnings.

## V6 overrides

### `import_urdf`

5.1 (current V5 adapter):
```python
status, import_config = omni.kit.commands.execute("URDFCreateImportConfig")
omni.kit.commands.execute("URDFParseFile", urdf_path=..., import_config=import_config)
omni.kit.commands.execute("URDFImportRobot", urdf_path=..., import_config=..., dest_path=...)
```

6.0 replacement:
```python
from isaacsim.asset.importer.urdf import URDFImporter, URDFImporterConfig
config = URDFImporterConfig(urdf_path=urdf_path, dest_path=prim_path, **kwargs)
importer = URDFImporter(config)
result_path = importer.import_urdf()
```

V6 override implements this. Return shape mirrors V5's existing return: `{"prim_path": result_path}` so handlers don't change.

### `_ensure_physics_world`

5.1 path (current `base.py` default — still works on 6.0 but uses deprecated `World`):
```python
from isaacsim.core.api import World
world = World.instance() or World(physics_dt=..., rendering_dt=..., stage_units_in_meters=1.0)
if world.physics_sim_view is None:
    world.initialize_physics()
```

6.0 replacement:
```python
from isaacsim.core.simulation_manager import SimulationManager
SimulationManager.setup_simulation(dt=1.0/60.0)  # idempotent
SimulationManager.initialize_physics()           # no-op if already initialized
```

V6 override calls `SimulationManager` first. If anything in `SimulationManager` raises (during the rc.59 window the API might still shift), fall through to the V5 default. This is the *only* method where we should diverge silently from the deprecation path, because `World` is the gateway used inside `ScriptNode` user code and we want that to stop emitting warnings on every `step_simulation` call.

### Deprecation noise suppression — only if needed

Plan: ship V6 first, run the smoke test, and check whether deprecation messages fire only once at first import (acceptable) or every adapter call (not acceptable). If the latter, add a `_silence_deprecation_warnings()` helper called from `IsaacAdapterV6.__init__` that uses `carb.log` filtering, applied scoped to module imports we control. Do not silence globally.

## Detection edge cases

- **Both 5.1 and 6.0 installed side-by-side** (user's current machine): the adapter picks whichever Isaac Sim runtime actually loaded the extension. `isaacsim.core.version.get_version()` reports the active runtime, so we're safe.
- **6.1 / future**: detection returns major ≥ 6 → use V6. Any specific 6.1 changes get a follow-up patch.
- **Detection failure**: log a `carb.log_warn` and fall back to V5. V5 is a strict subset of what works on 6.0, so the worst case is "deprecation warnings + correct behavior."

## Testing

Two layers:

1. **Adapter unit tests** (`tests/test_adapter_v6.py`): mock the import surface (`isaacsim.asset.importer.urdf`, `isaacsim.core.simulation_manager`) and assert the V6 overrides call the new APIs with the right arguments. Doesn't need Isaac Sim installed.
2. **Smoke test script** (`scripts/smoke_test_v6.py`): runnable inside Isaac Sim 6.0.0 itself — creates a robot, steps physics, checks no exceptions. This is the same smoke pattern the existing demo (`demo/franka_pick_place.py`) exercises, just headless. Documented in CONTRIBUTING but not in CI.

## Out of scope (followups)

- Migrating articulation handling to `isaacsim.core.experimental.prims.Articulation` (batch `wp.array` interface) — needs a wrapper layer that translates batched outputs back into single-robot numpy results, plus updates to handlers that currently assume single-prim adapters.
- Newton physics backend support — `SimulationManager.switch_physics_engine("newton")` is available but we keep PhysX as default.
- Replacing `isaacsim.sensors.camera.Camera` / `isaacsim.sensors.rtx.LidarRtx` with `isaacsim.sensors.experimental.rtx.{CameraSensor,LidarSensor,Lidar}` — different sensor-authoring vs sensor-runtime split needs its own design.
- Updating `extension.toml` dependencies — current deps still resolve in 6.0.

## Acceptance criteria

- `get_adapter()` returns `IsaacAdapterV6` when run inside Isaac Sim 6.0.0; `IsaacAdapterV5` when run inside 5.1.0.
- `mcp__isaac-sim__import_urdf` round-trips a URDF in 6.0.0 without printing `URDFCreateImportConfig is deprecated`.
- `play_simulation` followed by `step_simulation` in 6.0.0 runs without emitting `isaacsim.core.api.World deprecated` warnings on every step.
- All existing 5.1 behavior unchanged — V5 adapter file is not modified.
- `pyproject.toml` description and README updated to say "Isaac Sim 5.1.0 and 6.0.0".
