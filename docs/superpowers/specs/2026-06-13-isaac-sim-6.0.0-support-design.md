# Isaac Sim 6.0.0 Support — Design

## Goal

Make `isaacsim-mcp-server` run cleanly against Isaac Sim 6.0.0 while keeping 5.1.0 working. The existing `IsaacAdapterV5` already isolates version-specific calls behind an ABC, so most of the work is targeted: detect the runtime version, suppress / handle expected deprecation paths, and patch the calls that actually changed semantics.

## Context: what changed in 6.0.0

Isaac Sim 6.0.0 reorganizes the Python namespace around two ideas:

1. **A new "Core Experimental" API** (`isaacsim.core.experimental.{utils,prims,objects,actuators,materials}`) built on `warp` and `omni.physics.tensors`, intended to be the long-term replacement for `isaacsim.core.{api,prims,utils}`.
2. **A new `SimulationManager` singleton** (`isaacsim.core.simulation_manager`) that owns physics backend selection (`physx` vs `newton`) and supersedes `World` / `SimulationContext`.
3. **Newton physics as an opt-in alternative to PhysX**, shipped via a separate launcher (`isaac-sim.newton.sh` → `isaacsim.exp.full.newton.kit`). The default launcher (`isaac-sim.sh` → `isaacsim.exp.full.kit`) still uses PhysX — `isaacsim.core.simulation_manager/config/extension.toml:37` sets `exts."isaacsim.core.simulation_manager".default_engine = "physx"`. The Newton kit file explicitly **disables** all `omni.physx.*` modules and PhysX-dependent sensor extensions, then passes `--/exts/isaacsim.core.simulation_manager/default_engine=newton` and enables `isaacsim.physics.newton.auto_switch_on_startup = true`.

Critically, the 5.1 extensions still ship in 6.0.0 under `extsDeprecated/`:

| Extension used by V5 adapter | Status in 6.0.0 |
| --- | --- |
| `isaacsim.core.api` (`World`, `SimulationContext`) | Deprecated, still functional |
| `isaacsim.core.prims` (`SingleArticulation`, `SingleXFormPrim`) | Deprecated, still functional |
| `isaacsim.core.utils.{prims,stage,types}` | Deprecated, still functional |
| `isaacsim.sensors.camera.Camera` | Deprecated, still functional |
| `isaacsim.sensors.rtx.LidarRtx` | Deprecated, still functional |
| `isaacsim.storage.native.get_assets_root_path` | **Not deprecated** — unchanged |
| `omni.kit.commands.execute("URDFCreateImportConfig"/"URDFParseFile"/"URDFImportRobot")` | Registered, emits `carb.log_warn` per call |
| `omni.physx.get_physx_interface().get_rigidbody_transformation()` | **Unchanged** (verified in `omni.physx-110.1.11` `.pyi`) |
| `omni.kit.commands.execute("DeletePrims"/"CopyPrim"/"CreatePrim")` | Unchanged (standard kit commands) |
| `omni.usd`, `omni.timeline`, `omni.kit.app`, `omni.client`, `pxr.*` | Unchanged |

Each deprecated extension prints a one-line warning via `isaacsim.core.deprecation_manager` on first import. Under the **PhysX launcher** (the 6.0.0 default), nothing actually broke in the V5 adapter — it should run today, just noisy.

Under the **Newton launcher**, the V5 adapter breaks. Specifically:

- `omni.physx.get_physx_interface().get_rigidbody_transformation(prim_path)` — used in `IsaacAdapterV5.get_physics_state` and inside `step()` to read live rigid-body pose/velocity. The Newton kit disables `omni.physx` entirely, so this call cannot resolve.
- `isaacsim.sensors.{camera,rtx,physx}` — also disabled under Newton; affects `create_camera`, `capture_camera_image`, `create_lidar`, `get_lidar_point_cloud`.

`isaacsim.core.{api,prims,utils}` and `SimulationManager` are **backend-neutral** and work under both kits.

## Scope decision: minimal V6 adapter + Newton-aware physics calls

The "right" long-term direction is to migrate to `isaacsim.core.experimental.*`, but that API has fundamentally different semantics (batched `wp.array` over multi-prim views vs. single-prim `numpy` wrappers). A full rewrite is a bigger project than version support and would also chase still-shifting APIs in a 6.0 release tagged `-rc.59`.

This spec covers:

1. **Runtime version + engine detection** in `adapters/__init__.py:get_adapter()`.
2. **A new `IsaacAdapterV6(IsaacAdapterV5)`** that overrides only the calls where 5.1 behavior differs from 6.0 behavior:
   - `import_urdf` — migrate to `URDFImporter` / `URDFImporterConfig` (new public class) to avoid log spam and prepare for command removal.
   - `_ensure_physics_world` — prefer `SimulationManager.initialize_physics()` when present; fall back to `World.instance()` (the V5 path).
   - **`get_physics_state` and the rigid-body branch of `step`** — replace the direct `omni.physx.get_physx_interface().get_rigidbody_transformation()` call with a backend-neutral read via `SimulationManager.get_physics_simulation_view()` (the `omni.physics.tensors` view that both PhysX and Newton implement). Keeps the existing return shape so handlers don't change.
3. **Newton compatibility for the PhysX-tied paths.** Under the Newton launcher, `omni.physx` is not loaded at all, so PhysX-specific calls must not even attempt to import. V6's physics-state code goes through `omni.physics.tensors` instead; PhysX-only diagnostics (e.g. `omni.physx.get_physx_interface().get_contacts()` if we ever add it) get a clear `NotImplementedError("requires PhysX backend")`.
4. **Sensor handlers under Newton: explicit "not supported" error**, not a crash. The Newton kit disables `isaacsim.sensors.{camera,rtx,physx}` outright. The `create_camera` / `capture_camera_image` / `create_lidar` / `get_lidar_point_cloud` adapter methods detect this at call time and return a structured error so MCP clients see a readable message rather than an `ImportError` traceback. Full Newton sensor support (`isaacsim.sensors.experimental.rtx` and Newton's RTX bridge) is a follow-up.
5. **Suppress the expected deprecation noise** from the remaining deprecated imports — only if it turns out the warnings fire more than once at startup. Plan to ship V6 first, observe, then decide.
6. **A smoke test matrix** in `tests/` that imports each adapter against its expected runtime; gated behind an `ISAACSIM_VERSION` env var since CI won't have Isaac Sim installed.

Everything else (articulations, world, prim creation, materials, lighting) keeps using the 5.1-deprecated path under both PhysX and Newton — those code paths are backend-neutral.

## Architecture

```
adapters/
├── base.py              (unchanged — ABC)
├── v5.py                (unchanged — 5.1.0)
├── v6.py                (NEW — IsaacAdapterV6(IsaacAdapterV5))
└── __init__.py          (UPDATED — get_adapter() with version detection)
```

`get_adapter()` resolves the runtime version once at extension startup:

```python
def get_adapter() -> IsaacAdapterBase:
    major = _detect_isaacsim_major_version()  # returns 5 or 6
    if major == 6:
        from .v6 import IsaacAdapterV6
        return IsaacAdapterV6()
    from .v5 import IsaacAdapterV5
    return IsaacAdapterV5()
```

Version detection reads `isaacsim.core.version.get_version()` if available (present in both 5.1 and 6.0), with a fallback to reading the install's `VERSION` file via `carb.tokens` (e.g. `${app}/../VERSION`). On any failure, default to V5 — the safer choice since V5 also runs on 6.0 with warnings.

## V6 overrides

### `get_physics_state` and `step` rigid-body branch

5.1 (current V5 adapter at `v5.py:678`):
```python
import omni.physx
physx = omni.physx.get_physx_interface()
rb_data = physx.get_rigidbody_transformation(prim_path)
if rb_data and rb_data.get("ret_val", False):
    pos = rb_data["position"]
    vel = rb_data.get("linear_velocity", (0.0, 0.0, 0.0))
    ang = rb_data.get("angular_velocity", (0.0, 0.0, 0.0))
```

6.0 backend-neutral replacement:
```python
from isaacsim.core.simulation_manager import SimulationManager
view = SimulationManager.get_physics_simulation_view()
# omni.physics.tensors API: works for both PhysX and Newton
rb_view = view.create_rigid_body_view([prim_path])
transforms = rb_view.get_transforms()      # wp.array shape (1, 7) [x,y,z, qx,qy,qz,qw]
velocities = rb_view.get_velocities()      # wp.array shape (1, 6) [vx,vy,vz, wx,wy,wz]
```

V6 override implements this. We pay a small overhead per call (creating a view) — acceptable for an editor-level diagnostic tool, and these calls are not on a hot path. Caching the view keyed by prim_path is a follow-up if it shows up in profiles.

Fallback: if `SimulationManager.get_physics_simulation_view()` returns `None` (simulation hasn't been initialized yet), fall back to reading the USD-authored transform via `get_prim_transform` — same fallback V5 already uses.

### `import_urdf`

5.1 (current V5 adapter):
```python
status, import_config = omni.kit.commands.execute("URDFCreateImportConfig")
omni.kit.commands.execute("URDFParseFile", urdf_path=..., import_config=import_config)
omni.kit.commands.execute("URDFImportRobot", urdf_path=..., import_config=..., dest_path=...)
```

6.0 replacement:
```python
from isaacsim.asset.importer.urdf import URDFImporter, URDFImporterConfig
config = URDFImporterConfig(urdf_path=urdf_path, dest_path=prim_path, **kwargs)
importer = URDFImporter(config)
result_path = importer.import_urdf()
```

V6 override implements this. Return shape mirrors V5's existing return: `{"prim_path": result_path}` so handlers don't change.

### `_ensure_physics_world`

5.1 path (current `base.py` default — still works on 6.0 but uses deprecated `World`):
```python
from isaacsim.core.api import World
world = World.instance() or World(physics_dt=..., rendering_dt=..., stage_units_in_meters=1.0)
if world.physics_sim_view is None:
    world.initialize_physics()
```

6.0 replacement:
```python
from isaacsim.core.simulation_manager import SimulationManager
SimulationManager.setup_simulation(dt=1.0/60.0)  # idempotent
SimulationManager.initialize_physics()           # no-op if already initialized
```

V6 override calls `SimulationManager` first. If anything in `SimulationManager` raises (during the rc.59 window the API might still shift), fall through to the V5 default. This is the *only* method where we should diverge silently from the deprecation path, because `World` is the gateway used inside `ScriptNode` user code and we want that to stop emitting warnings on every `step_simulation` call.

### Deprecation noise suppression — only if needed

Plan: ship V6 first, run the smoke test, and check whether deprecation messages fire only once at first import (acceptable) or every adapter call (not acceptable). If the latter, add a `_silence_deprecation_warnings()` helper called from `IsaacAdapterV6.__init__` that uses `carb.log` filtering, applied scoped to module imports we control. Do not silence globally.

## Detection edge cases

- **Both 5.1 and 6.0 installed side-by-side** (user's current machine): the adapter picks whichever Isaac Sim runtime actually loaded the extension. `isaacsim.core.version.get_version()` reports the active runtime, so we're safe.
- **6.1 / future**: detection returns major ≥ 6 → use V6. Any specific 6.1 changes get a follow-up patch.
- **Detection failure**: log a `carb.log_warn` and fall back to V5. V5 is a strict subset of what works on 6.0 PhysX, so the worst case is "deprecation warnings + correct behavior."
- **Active engine = newton**: V6 adapter detects this via `SimulationManager.get_active_physics_engine()` at construction time and stores it on `self._engine`. PhysX-only methods (sensor creation, sensor reads) check this flag and return a structured `{"status": "error", "message": "..."}` instead of attempting the import.

## Testing

Two layers:

1. **Adapter unit tests** (`tests/test_adapter_v6.py`): mock the import surface (`isaacsim.asset.importer.urdf`, `isaacsim.core.simulation_manager`) and assert the V6 overrides call the new APIs with the right arguments. Doesn't need Isaac Sim installed.
2. **Smoke test script** (`scripts/smoke_test_v6.py`): runnable inside Isaac Sim 6.0.0 itself — creates a robot, steps physics, checks no exceptions. This is the same smoke pattern the existing demo (`demo/franka_pick_place.py`) exercises, just headless. Documented in CONTRIBUTING but not in CI.

## Out of scope (followups)

- Migrating articulation handling to `isaacsim.core.experimental.prims.Articulation` (batch `wp.array` interface) — needs a wrapper layer that translates batched outputs back into single-robot numpy results, plus updates to handlers that currently assume single-prim adapters.
- **Full Newton sensor support** — `isaacsim.sensors.experimental.rtx.{CameraSensor,LidarSensor,RtxCamera,Lidar}` is the Newton-compatible sensor API (different sensor-authoring vs sensor-runtime split). V6 returns "not supported under Newton" errors today; a follow-up spec wires up the experimental sensor API.
- **Replacing the deprecated `isaacsim.sensors.camera.Camera` / `isaacsim.sensors.rtx.LidarRtx`** in the PhysX path — works fine for now, can migrate alongside the Newton sensor work.
- Updating `extension.toml` dependencies — current deps still resolve under the PhysX kit in 6.0. Under the Newton kit, the deps on `isaacsim.sensors.*` would fail to enable; addressed when sensor support lands.

## Acceptance criteria

- `get_adapter()` returns `IsaacAdapterV6` when run inside Isaac Sim 6.0.0; `IsaacAdapterV5` when run inside 5.1.0.
- `mcp__isaac-sim__import_urdf` round-trips a URDF in 6.0.0 PhysX without printing `URDFCreateImportConfig is deprecated`.
- `play_simulation` followed by `step_simulation` in 6.0.0 PhysX runs without emitting `isaacsim.core.api.World deprecated` warnings on every step.
- `get_physics_state` returns the correct rigid-body pose and velocity under **both** the PhysX kit (`isaac-sim.sh`) and the Newton kit (`isaac-sim.newton.sh`).
- Sensor tools (`create_camera`, `capture_image`, `create_lidar`, `get_lidar_point_cloud`) called under the Newton kit return `{"status": "error", "message": "Sensor X is not supported under Newton..."}` instead of crashing with an `ImportError`.
- All existing 5.1 behavior unchanged — V5 adapter file is not modified.
- `pyproject.toml` description and README updated to say "Isaac Sim 5.1.0 and 6.0.0" and note that the Newton launcher has reduced sensor support.
